                <p>Ultimately, you run your Analysis Engine from within your application, but it helps to be able to test it independently.  Fortunately, the UIMA SDK includes tools for just such an occasion.</p>

                <p><a name="N106D5"><span class="smalltitle">Start the CAS Visual Debugger</span></a></p>
                <p>The UIMA SDK actually comes with several tools you can use to test your Analysis Engine, but to really see what's going on, use the CAS Visual Debugger, which lets you see exactly what Annotations are being added to the CAS object.  To start the debugger, do the following:</p>
                <ol>
                    <li>In the main Eclipse window, select Run &gt; Run.</li>

                    <li>Expand the Java node and select UIMA CAS Visual Debugger. See <a href="section8.html#fig15">Figure 15</a>.<br />

                        <br />
                        
                            <br /><a name="fig15"><b>Figure 15. Starting the CAS Visual Debugger</b></a><br />
                            <img alt="Starting the CAS Visual Debugger" height="515" src="fig15.jpg" width="572"/>
                        <br />
                    </li>
                    <li>Click Run</li>
                </ol>
                <br /><table border="0" cellspacing="0" cellpadding="0" width="100%"><tr><td><img width="100%" src="//www.ibm.com/i/v14/rules/blue_rule.gif" height="1" alt=""/></td></tr></table><table class="no-print" cellspacing="0" cellpadding="0" align="right"><tr align="right"><td><table border="0" cellpadding="0" cellspacing="0"><tr><td valign="middle"><img width="16" src="//www.ibm.com/i/v14/icons/u_bold.gif" height="16" border="0" alt=""/><br /></td><td valign="top" align="right"><a href="#main" class="fbox"><b>Back to top</b></a></td></tr></table></td></tr></table><br /><br /><p><a name="N10700"><span class="smalltitle">Specify the document</span></a></p>

                <p>The next step is to specify the text you want to analyze.  The CAS Visual Debugger, shown in <a href="section8.html#fig16">Figure 16</a>, lets you enter text directly or load a text file.  Click <b>File &gt; Open text file</b> and specify the text version of the company report you want to analyze.</p>
                
                    <br /><a name="fig16"><b>Figure 16. Specifying the document to analyze</b></a><br />
                    <img alt="Specifying the document to analyze" height="474" src="fig16.jpg" width="572"/>
                <br />

                <br />
                <br />
                <p>Click <b>Open</b> to load the document.</p>
                <br /><table border="0" cellspacing="0" cellpadding="0" width="100%"><tr><td><img width="100%" src="//www.ibm.com/i/v14/rules/blue_rule.gif" height="1" alt=""/></td></tr></table><table class="no-print" cellspacing="0" cellpadding="0" align="right"><tr align="right"><td><table border="0" cellpadding="0" cellspacing="0"><tr><td valign="middle"><img width="16" src="//www.ibm.com/i/v14/icons/u_bold.gif" height="16" border="0" alt=""/><br /></td><td valign="top" align="right"><a href="#main" class="fbox"><b>Back to top</b></a></td></tr></table></td></tr></table><br /><br /><p><a name="N10728"><span class="smalltitle">Specify the Analysis Engine descriptor</span></a></p>
                <p>Once you've got the text, you need to specify the Analysis Engine.  Before you can do that, however, you need to compensate for some classpath issues.  Even though you set the data path to point to the descriptors folder when you imported the type system into the descriptor, the CAS Visual Debugger won't know to look for it there, looking instead in the resources directory.</p>

                <p>To solve the problem, copy the typeSystemDescriptor.xml to the resources directory.  Right-click the file and choose Copy, then right-click the resources folder and choose Paste.</p>
                <p>Now you can load the Analysis Engine as follows:</p>
                <ol>
                    <li>In the CAS Visual Debugger, select Run &gt; Load TAE.  (TAE stands for Text Analysis Engine.)</li>
                    <li>Navigate to the ProductNumberAEDescriptor.xml file and click <b>Open</b>.</li>

                </ol>
                <br /><table border="0" cellspacing="0" cellpadding="0" width="100%"><tr><td><img width="100%" src="//www.ibm.com/i/v14/rules/blue_rule.gif" height="1" alt=""/></td></tr></table><table class="no-print" cellspacing="0" cellpadding="0" align="right"><tr align="right"><td><table border="0" cellpadding="0" cellspacing="0"><tr><td valign="middle"><img width="16" src="//www.ibm.com/i/v14/icons/u_bold.gif" height="16" border="0" alt=""/><br /></td><td valign="top" align="right"><a href="#main" class="fbox"><b>Back to top</b></a></td></tr></table></td></tr></table><br /><br /><p><a name="N10743"><span class="smalltitle">Run the debugger</span></a></p>

                <p>Finally, it's time to test the Analysis Engine.  In the CAS Visual Debugger window, select Run &gt; Run ProductNumberAEDescriptor.  The results appear in the upper left-hand section of the window, under Analysis Results. See <a href="section8.html#fig17">Figure 17</a>.  If this pane is too small to see the results, you can drag its borders to expand it.</p>
                
                    <br /><a name="fig17"><b>Figure 17. Specifying the document to analyze</b></a><br />

                    <img alt="Specifying the document to analyze" height="472" src="fig17.jpg" width="572"/>
                <br />
                <br />
                <br />
                <p>Expand the <code>AnnotationIndex</code> node until you see the <code>ProductNumber</code> Annotations.  With this document, there should be three.  (The <code>DocumentAnnotation</code> Annotation refers to the document itself.)  If you select the <code>ProductNumber</code> Annotation, you see all of the Annotations of that type in the lower pane.</p>

                <p>Click each of the Annotations to see them highlighted in the actual document.  You can also expand the Annotation to see its features, such as its starting and ending position, and the <code>productLine</code>.</p>
                <p>Now that you know your Analysis Engine works, it's time to incorporate it into the actual application.</p>
